Quick Start Guide

Ready-to-use STRXplorer with pre-loaded data

🎉 Great News!

This repository comes with all GWAS data pre-loaded!
No downloading, no waiting - just clone and run for instant access to all Manhattan plots.

Step 1: Quick Setup (2 minutes)

Get up and running in just a few commands:


# Clone the repository
git clone https://github.com/ciarareeve/STRXplorer
cd STRXplorer

# Install Python dependencies
pip install flask pandas numpy plotly sqlite3

# Run the application
python app.py
That's it! The application will start at http://localhost:5000 with full functionality.

Step 4: System Requirements

Minimal requirements for running the platform:

  • Python 3.8 or higher
  • 4GB RAM (8GB recommended for large datasets)
  • 2GB free disk space
  • Modern web browser (Chrome, Firefox, Safari, Edge)

Required Python packages:


flask          # Web framework
pandas         # Data manipulation
numpy          # Numerical computing
plotly         # Interactive plotting
sqlite3        # Database operations (usually built-in)
💡 Note: SQLite3 is typically included with Python installations. If you encounter issues, install it with pip install pysqlite3

Step 5: Pre-loaded Data Overview

The databases in the data/ directory contain:

manhattan_data.db:

  • Multiple traits: Blood traits, anthropometric measures, and more
  • Millions of variants: Genome-wide association data
  • Statistical summaries: P-values, effect sizes, confidence intervals
  • Optimized indexes: Fast querying for real-time plotting

locus_data.db:

  • STR locus information: Genomic coordinates and repeat motifs
  • Allele-phenotype data: Individual allele effect sizes
  • Sample counts: Number of individuals per allele
  • Confidence intervals: Statistical uncertainty estimates
💡 Data Source: GWAS data from Margoliash et al. 2023 study on STR associations with complex traits.

Step 6: Troubleshooting

Common Issues & Quick Fixes:

Application won't start:
• Check Python version: python --version (should be 3.8+)
• Install missing packages: pip install flask pandas numpy plotly
• Ensure you're in the STRXplorer root directory
• Check if port 5000 is already in use
Database errors:
• Verify both data/locus_data.db and data/manhattan_data.db exist
• Check file permissions (should be readable)
• Ensure the data/ directory structure is intact
• Try running from the repository root directory
No plots showing:
• Check the Database Status page
• Verify JavaScript is enabled in your browser
• Try refreshing the browser
• Clear browser cache if plots appear corrupted
• Check browser console for error messages (F12 → Console)
Port already in use:
• Kill existing Flask processes: pkill -f flask or pkill -f python
• Or specify a different port: Edit app.py and change app.run(port=5001)
• Find what's using port 5000: lsof -i :5000 (Mac/Linux) or netstat -ano | findstr :5000 (Windows)
Import errors:
• Make sure you're running Python from the STRXplorer directory
• Check if the src/ directory structure is intact
• Verify all __init__.py files exist (if any)
• Try running with python -m flask run instead

Step 7: Advanced Usage (Optional)

Configuration:

Modify config.py to customize database paths and other settings:


# Example config.py modifications
class Config:
    LOCUS_DB_PATH = "data/locus_data.db"           # STR locus database
    MANHATTAN_DB_PATH = "data/manhattan_data.db"   # GWAS data database
    DEBUG = True                                   # Enable debug mode
    PORT = 5000                                    # Change default port

Database Maintenance:


# Optional: Check database integrity
sqlite3 data/manhattan_data.db
.schema
SELECT COUNT(*) FROM gwas_variants;
.exit

# Check STR locus data
sqlite3 data/locus_data.db
.schema  
SELECT COUNT(*) FROM locus_data;
.exit

API Usage:

Access data programmatically through the API endpoints:


# Get list of available traits
curl http://localhost:5000/api/trait_list

# Get database status information
curl http://localhost:5000/database_status_json
